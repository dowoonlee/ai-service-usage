## 0. TL;DR

- 현재 맵은 **"클릭 가능한 장식"** — 진행(2/8)이 숫자로만 존재하고 맵 자체는 아무것도 기억하지 못한다. 업적 시스템의 맵이 가져야 할 세 가지(여정·발견·흔적)가 모두 없다.
- 10개 게임의 맵 문법을 검토한 결과, 우리 조건(수동 지표·glance UI·픽셀 자산 보유)에 맞는 것은 **노드-루트 오버월드(SMB3) + 구름 안개 탐험(Civ/Zelda) + 테라포밍(동물의 숲)**의 하이브리드. 실시간 조작류(StS 분기 선택, PoE 아틀라스)는 부적합.
- 구조 제안: 대륙 전환 UI 대신 **단일 연속 월드 + 2단 카메라(월드 뷰 ↔ 지역 뷰)**. Cloud 대륙은 "다른 페이지"가 아니라 바다 건너 실제 좌표에 존재하고, 미발견 지역은 **구름에 덮여 있다**(테마와 메커니즘의 일치).
- 펫이 맵 위를 실제로 걸어다니는 **트레이너 아바타**(트레이너 카드 아바타와 동기화), 도장 건물이 tier 진행에 따라 **천막→도장→성으로 성장**, 클리어한 지역엔 영구 흔적. 전부 기존 자산(PetSprite·Overworld.png 건물·WeatherParticles)으로 구현 가능.
- 종착점(north star): 맵이 도장 전용 장식을 넘어 **앱 게이미피케이션의 허브** — Arena 마을 클릭 = 아레나 탭, Guild 요새 클릭 = 길드 탭.

## 1. 현재 맵의 문제 정의

`WorldMapView.swift`의 현 구조를 냉정하게 평가하면:

| 관점 | 현재 상태 | 문제 |
|---|---|---|
| **여정(Journey)** | 5 region이 처음부터 전부 노출, 순서·경로 없음 | "어디까지 왔고 어디로 가는지"가 없다. 진행은 `2/8` 숫자뿐 |
| **발견(Discovery)** | 첫 방문에 모든 것이 보임 | 새 콘텐츠 추가가 "지도 어딘가 마커 하나 늘어남"으로만 체감 |
| **흔적(Permanence)** | 클리어해도 맵은 불변 (마커 숫자만 변함) | 뱃지 32개를 깨도 맵은 첫날과 동일 — 업적의 물리적 보상 부재 |
| **생동감(Liveness)** | 정적. hover dim이 유일한 반응 | 걷는 펫·날씨 파티클 자산이 있는 앱인데 맵만 죽어 있음 |
| **확장성(Scale)** | tiles 매트릭스+regionPositions 하드코딩 | region 추가 = 코드 수정. Voronoi가 전 대륙 재분배됨 |

요약: **지도가 상태(state)를 갖지 않는다.** 업적 시스템의 지도는 "유저가 이 앱과 보낸 시간의 지형도"여야 하는데, 지금은 카테고리 선택용 라디오 버튼의 스킨이다.

## 2. 선행연구 — 10개 게임의 맵 문법

각 게임에서 **훔칠 것**과 **버릴 것**을 명시한다. (평가 축: 수동 지표 적합성 / 528×220pt glance UI 적합성 / 기존 자산 재사용성)

### 2.1 Pokémon (본가 시리즈 타운맵) — 원류 메타포
- **문법**: 마을(도장)들이 루트로 연결된 지역. 뱃지가 진행 게이트(HM 잠금). 타운맵은 "가본 곳"의 기록이자 자랑거리. 최종 관문(챔피언로드)이 지리적으로 존재.
- **훔칠 것**: ① 도장=마을이라는 공간화 ② "방문한 마을 / 아직 이름만 아는 마을"의 시각 구분 ③ 챔피언의 지리적 실체(맵 중앙 or 최북단의 리그 건물).
- **버릴 것**: HM식 하드 게이팅 — 우리 지표는 병렬 누적이라 순서 강제는 기존 유저 진행과 충돌.

### 2.2 Super Mario Bros 3 / World — 노드-패스 오버월드
- **문법**: 레벨=노드, 클리어하면 다음 경로 구간이 **그려지며** 열림. 분기·비밀 출구(SMW 스타로드)·맵 자체의 영구 변화(스위치 궁전).
- **훔칠 것**: ① **경로가 그려지는 연출** — tier 클리어마다 다음 노드까지 길이 이어짐. 작은 화면에서 진행을 보여주는 최적 문법 ② 비밀 노드(이스터에그 뱃지 자리) ③ 클리어 노드의 색 변화.
- **버릴 것**: 스테이지 선택 커서 조작 — 우리는 노드를 "플레이"하지 않음.
- **적합성 평가**: 노드-패스는 **작은 패널에서 가장 정보 밀도가 높은 진행 표현**. 528×220에 마을 9개+경로면 충분히 성립.

### 2.3 Candy Crush 등 사가맵 — 무한 스크롤 선형 경로
- **문법**: 수백 노드가 굽이치는 한 줄 길. 테마 존이 N개마다 바뀜. 스크롤이 곧 진행 자랑.
- **훔칠 것**: **"존이 계속 이어진다"는 확장 문법** — 대륙 전환 페이지가 아니라 지도가 옆으로 계속 자라는 모델. 신규 콘텐츠 = 지도 끝에 새 존.
- **버릴 것**: 완전 선형 — 우리 카테고리는 병렬 진행이므로 큰 줄기만 빌리고 지역 내부는 자유.

### 2.4 Civilization / Zelda 계열 — 전장의 안개(Fog of War)
- **문법**: 지도가 어둠/구름으로 덮여 시작, 도달해야 드러남. 발견 자체가 도파민.
- **훔칠 것**: **구름 안개**. 우리 제2 대륙 테마가 "Cloud"라서 안개=구름이 **테마와 메커니즘의 완전한 일치**가 된다. 미발견 지역은 픽셀 구름 아래 실루엣만, 해금 시 구름이 갈라지는 연출.
- **버릴 것**: 타일 단위 정밀 탐사 — 지역 단위 해금이면 충분. 타일 단위는 관리 비용 대비 과함.

### 2.5 Metroidvania (Zelda/Metroid) — 능력 게이팅과 재방문
- **문법**: 보이지만 못 가는 곳. 능력을 얻고 돌아오면 열림.
- **훔칠 것**: **소프트 게이트의 문법만** — "Cloud 대륙행 다리는 mainland 뱃지 4개부터" 같은 잠금을 지리적 오브젝트(끊긴 다리·항구)로 표현. 잠금 사유가 지도 위에 보이는 것이 핵심.
- **버릴 것**: 백트래킹 강요.

### 2.6 동물의 숲 / Stardew Valley — 성장하는 정주지(테라포밍)
- **문법**: 지도는 가꾸는 장소. 건물이 생기고 업그레이드되고, 마을이 유저 히스토리의 물리적 축적물이 됨.
- **훔칠 것**: **이 기획의 심장.** ① 도장 건물 성장: 공터→천막(localhost)→목조 도장(dev)→석조 홀(staging)→성(production) ② 지역 마스터 시 바이옴 상태 변화(용암 지대가 식어 흑요석 평원으로, 빙판에 오로라) ③ 챔피언 기념비 ④ **보유 펫이 해금 지역을 배회하는 주민화** — 가챠 인벤토리가 지도에 살게 됨.
- **자산 검증**: 이미 실물 확인한 `Overworld.png`에 집·천막·시장 가판·석상·우물 스프라이트 존재, `Caves Etc.png`에 크리스탈·다리·석상 존재 → **건물 4단계를 외부 에셋 없이 조립 가능**.
- **버릴 것**: 유저 직접 배치/커스터마이징 — 조작 UI 비용 큼. 성장은 전부 지표 자동.

### 2.7 Slay the Spire / FTL — 절차적 분기 선택
- **문법**: 매 런마다 새 분기 맵, 경로 선택이 곧 전략.
- **평가**: **부적합.** 선택-결과 루프는 능동 플레이 전제. 우리 지표는 백그라운드 누적이라 "선택"이 의미를 갖기 어려움. 다만 "시즌 원정 이벤트" 같은 별도 모드의 소재로 보류.

### 2.8 PoE 아틀라스 / 원신 지역 개척도 — 메타 맵 진행
- **문법**: 지역별 개척도%·상호 강화 노드망.
- **훔칠 것**: **지역 개척도 개념만** — region 카드에 "Coffee 지방 개척도 75%" (뱃지+발견+건물 종합). 트레이너 카드·랭킹에 실을 수 있는 단일 수치.
- **버릴 것**: 노드 상호작용 최적화 — 복잡도 폭발.

### 2.9 GitHub 잔디 (contribution graph) — 개발자 네이티브 "지도"
- **문법**: 시간을 지형처럼 렌더. 활동 강도=색 농도. 개발자가 이미 학습한 시각 언어.
- **훔칠 것**: **살아있는 지면** — 최근 7일 사용량에 비례해 해금 지역 풀 색이 미묘하게 무성/시듦. 사용량 추적 앱의 지도가 실제 사용량을 반영한다는 것은 **이 앱만 할 수 있는 차별점**. (과하면 픽셀 톤 훼손 — 채도 ±10% 이내 서브틀 오버레이로.)
- **추가**: `WeatherAPI`+`WeatherParticles`가 이미 있으므로 **실제 날씨를 맵 위에** — 비 오는 날 지도에도 비가 온다. 코드 재사용 그대로.

### 2.10 Mario Party / 스고로쿠 보드 — 말이 걸어다니는 판
- **문법**: 아바타(말)가 보드를 물리적으로 이동. 이동 자체가 내러티브.
- **훔칠 것**: **트레이너 아바타** — 트레이너 카드의 대표 펫이 맵 루트를 실제로 걸어다님(`PetSprite` walk 프레임 재사용, `WalkingCat` 이동 로직 이식). 위치는 "현재 가장 진행 중인 도장 방향으로 이동 중" — 수동 지표를 여정 내러티브로 번역하는 장치. 클릭하면 `Quotes` 한마디.
- **버릴 것**: 주사위/칸 이벤트 — 조작 게임화 금지 원칙.

## 3. 디자인 필러 (판단 기준)

1. **수동-우선(Passive-first)**: 유저가 아무것도 안 해도 맵이 변한다. 조작이 필요한 재미는 전부 부차.
2. **5초 가독성**: 패널을 연 순간 "내가 어디까지 왔나"가 숫자 없이 그림으로 보인다.
3. **모든 업적은 흔적을 남긴다**: 뱃지 하나 = 맵의 픽셀 하나라도 영구히 바뀜.
4. **콘텐츠 = 데이터**: 지역 추가가 코드가 아니라 데이터 블록 추가.
5. **자산 재사용 우선**: PetSprite / Intersect 시트 / WeatherParticles / Quotes. 신규 에셋은 픽셀 구름 정도.

**안티-골**: 실시간 조작 게임화 ✕ / 그라인드 강요 ✕ (사용량은 업무의 부산물이어야) / 유저 배치·꾸미기 UI ✕ (트레이너 카드가 이미 담당).

## 4. 제안 구조 — "구름 걷히는 하나의 세계"

### 4.1 월드 구조: 단일 연속 월드 + 2단 카메라

`gym-expansion.md` §4.1의 "대륙 전환(페이지 스위치)" 결정을 **폐기 제안**. 대신:

- **가상 월드 좌표계 하나** (예: 96×40 셀). mainland는 서쪽, Cloud 제도는 바다 건너 동쪽. 물리적으로 같은 지도.
- **월드 뷰** (기본): 전 세계를 축소 렌더. 마을 아이콘·구름 덮인 미지 영역·아바타 위치가 보임. 카테고리 상세는 안 보임 — "내 세계의 전경".
- **지역 뷰**: region 클릭 → 카메라가 해당 지역으로 **팬+줌**(0.25s ease). 타일 디테일·도장 건물·관장·배회 펫이 보이는 스케일. 기존 하단 카테고리 그리드는 지역 뷰와 동기.
- 대륙 전환 UI가 사라짐 — 새 대륙은 "지도 동쪽에 원래 있던, 구름에 덮여 있던 땅". 콘텐츠 추가가 **세계의 발견**으로 체감된다(2.3·2.4 문법).

기술적으로: Canvas는 어차피 viewport만 그리므로 월드가 3배 커져도 렌더 비용 동일. `WorldMapDesign`의 정적 매트릭스를 아래 데이터 모델로 대체.

### 4.2 데이터 모델 (콘텐츠=데이터 원칙)

```swift
struct WorldMap: Codable {          // JSON 리소스 or Swift 상수 — 지역 추가는 이 데이터만
    let size: (cols: Int, rows: Int)
    let terrain: [TerrainLayer]      // 바이옴별 셀 집합 + 타일 ref
    let towns: [Town]                // 도장 마을
    let routes: [Route]              // 마을 간 경로 폴리라인
    let landmarks: [Landmark]        // 기념비·다리·항구(게이트 오브젝트)
}
struct Town { let region: BadgeRegion; let cell: Cell; /* 건물 스테이지는 파생 */ }
struct Route { let from, to: BadgeRegion; let cells: [Cell] }  // 경로가 그려질 칸들
```

파생 상태(저장 안 함): 건물 스테이지 = region 내 최고 tier, 경로 점등 = 인접 마을 뱃지 수, 바이옴 상태 = 지역 마스터 여부.
영속 상태(신규, 최소): `Settings.discoveredRegions: Set<String>` 하나. 나머지는 전부 기존 카운터에서 파생 — **세이브 마이그레이션 리스크 최소화**.

### 4.3 핵심 시스템 4개

**A. 루트와 경로 점등 (SMB3)**
마을 사이 길은 처음엔 희미한 점선. 출발 마을에서 tier를 깰 때마다 경로 타일이 한 구간씩 **밟은 길**로 바뀌고, 4/4이면 다음 마을까지 완전히 이어짐. 병렬 진행 원칙 유지 — 길이 안 이어져도 다른 지역 진행은 자유. 길은 **순서의 강제가 아니라 여정의 기록**.

**B. 구름 발견 (Civ×테마)**
- 신규 유저: mainland의 Coffee만 맑음, 나머지는 옅은 구름(실루엣 보임 — "저기 뭔가 있다"). 지역에 뱃지 1개 생기면 그 지역 구름이 갈라지는 연출.
- Cloud 제도: 짙은 구름 + 바다. mainland 뱃지 N개(예: 8) 도달 시 항구에 배가 생기고 제도 방향 구름이 걷히기 시작 — **소프트 게이트를 지리 오브젝트로 표현**(2.5).
- 기존 유저 마이그레이션: 진행 흔적이 있는 지역은 전부 발견 처리. 챔피언은 전 지도 맑음 + 기념비.

**C. 테라포밍 (동숲) — 업적의 물리적 축적**
| 트리거 | 맵 변화 |
|---|---|
| tier 클리어 | 도장 건물 성장 (공터→천막→목조→석조→성) |
| 지역 마스터 | 바이옴 상태 전이 (용암→흑요석 평원 / 빙판에 오로라 / 사막에 오아시스 / 암반 요새에 깃발) |
| 대륙 챔피언 | 대륙 중앙 기념비 + 관장들이 기념비 앞에 모임(sit 스프라이트) |
| 컬렉션 컴플리트 | 해당 지역에 소품 추가 (석상·우물·가판 — Overworld.png 소스) |

**D. 살아있는 지도 (생동감 레이어)**
- **트레이너 아바타**: 트레이너 카드 대표 펫이 루트 위를 걷는다. 목적지 = 현재 가장 활발히 진행 중인 카테고리의 마을(최근 7일 지표 증가율 기준). 도착해 있으면 마을 앞에서 idle. 클릭 시 Quotes.
- **주민 펫**: 발견된 지역마다 보유 펫 1~2마리 랜덤 배회(하루 단위 로테이션) — 가챠 인벤토리의 지도 노출.
- **날씨**: `WeatherAPI` 현재 날씨를 맵에 그대로(`WeatherParticles` 재사용). Cloud 제도는 항상 약한 안개 파티클.
- **잔디 농도**: 최근 7일 코인 적립량에 비례해 발견 지역 풀 채도 ±10% (후순위).

### 4.4 인터랙션 모델 (전부 기존 문법 유지·확장)

| 입력 | 반응 |
|---|---|
| 월드 뷰에서 region 클릭 | 지역 뷰로 팬+줌, 하단 카테고리 그리드 동기 (현행 유지) |
| 지역 뷰에서 빈 곳/뒤로 클릭 | 월드 뷰 복귀 |
| 마을 건물 hover | 개척도% + 다음 성장 조건 popover (현행 tooltip 패턴) |
| 아바타/배회 펫 클릭 | 말풍선 한마디 |
| 구름 지역 클릭 | "구름 너머에 무언가 있다" + 해금 조건 힌트 |

높이 220pt 유지 가능(월드 뷰가 축소 렌더이므로). 지역 뷰에서 240pt로 살짝 확장은 선택.

### 4.5 North star — 게이미피케이션 허브

장기적으로 마을=기능 진입점: Arena 투기장 건물 클릭 → 아레나 탭, Guild 요새 → 길드 탭, Daily 사원 → 데일리 퀴즈. 맵이 "도장 탭의 장식"에서 **앱 세계관의 홈 화면**으로 승격. 이번 스코프엔 미포함, 구조(Town에 `deepLink` 필드 여지)만 열어둠.
