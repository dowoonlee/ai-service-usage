# 포켓몬 골드/실버/크리스탈(Gen 2, GBC) 배틀 화면 UI 레이아웃 조사

## 목적
AIUsage(도트 감성 macOS 앱)의 펫 배틀 화면을 설계할 때 참고할 **기능적 레퍼런스**. 포켓몬의 저작권 그래픽(스프라이트·폰트·팔레트)을 복제하지 않고, **화면 배치·UI 구조·색 임계값·애니메이션 관례**만 사실로 추출한다. 실제 이미지 재현 목적 아님.

## 요약 (3줄)
- Gen 2 배틀 화면은 GB 해상도 160×144px(20×18 타일, 8px 타일 기준) 위에 상대(오른쪽 위, 정면 스프라이트)·내 포켓몬(왼쪽 아래, 뒷모습 56×56px 캔버스) 두 스프라이트와 각각의 HP 정보 박스(상대=좌상단, 나=우하단), 하단 텍스트/커맨드 박스로 구성된다.
- HP바 색 임계값은 **Gen2부터 퍼센트 기반**: 초록(>50%) → 노랑(20~50%) → 빨강(<20%). 정확히 절반(50%)으로 회복 시 Gen2 한정으로 초록 유지되는 quirk가 있다. [출처: Bulbapedia HP]
- 파티 상태는 포켓볼 아이콘(정상=빨강/흰 볼, 상태이상/기절=검게 표시)으로 간접 확인되나, 이 자료는 웹서치 요약 기반이라 Bulbapedia 원문 직접 인용은 확보하지 못했다 — **확인 안 됨(약함)**으로 표시.

## 권장 결론 (트레이드오프 명시)
- 화면 2분할(상대 우상단 / 나 좌하단) + HP박스 대각선 배치는 **매우 잘 확립된 관례**이므로 그대로 차용해도 저작권 리스크가 낮다(레이아웃/구조는 저작권 보호 대상이 아니며, 우리는 오리지널 그래픽으로 그린다는 전제). 다만 "완전히 동일한 비율·타일 좌표"까지 그대로 베끼기보다는 **기능(무엇이 어디에 있는가)만 참고**하고 실제 픽셀 좌표는 우리 앱의 캔버스 크기에 맞게 재설계 권장.
- HP바 색 임계값(50%/20%)은 업계 표준처럼 널리 쓰이는 UX 관례이므로 그대로 채용해도 무방. 우리 코드베이스에 이미 색상 임계값 로직이 있다면(예: 사용량 게이지) 그것과 통일 여부를 결정할 것.
- 기절 시 "아래로 슬라이드하며 사라짐" + 피격 시 "스프라이트 반전/깜빡임"은 Gen1부터 이어진 전 세대 공통 관례이므로 재구성 시 리스크 없이 차용 가능.

## 사실 (출처별)

### 1. 전체 화면 구성 / 스프라이트 위치

- GB/GBC 화면 해상도는 **160×144px**, 8×8px 타일 기준 **20×18 타일** 그리드. [출처: Bulbapedia HP / 일반적으로 알려진 GB 하드웨어 스펙, 별도 링크 없음 — 하드웨어 스펙 자체는 업계 공지 사실]
- 스프라이트 캔버스 크기(Gen 1·2 공통): **정면(front) 스프라이트**는 크기별로 소(5×5타일=40×40px) / 중(6×6타일=48×48px) / 대(7×7타일=56×56px). **뒷모습(back) 스프라이트**는 Gen1은 7×7타일(56×56px), Gen2는 56×56px 영역이 예약되어 있으나 실제 그림은 48×48px만 사용. [출처: Bulbapedia, "Pokémon sprite" — https://bulbapedia.bulbagarden.net/wiki/Pok%C3%A9mon_sprite]
- 스프라이트 정렬 규칙: 소형 정면 스프라이트(5×5)는 캔버스 중앙-하단 정렬(위 2타일, 좌우 각 1타일 여백). 중형 정면 스프라이트(6×6) 및 Gen2 뒷모습 스프라이트는 캔버스 우측-하단 정렬(위 1타일, 좌측 1타일 여백). [출처: Bulbapedia, "Pokémon sprite"]
- Gen1·2 공통으로 정면/뒷모습 스프라이트 배경은 **흰색, 투명도 없음**(전용 배경 그래픽 없이 순수 흰 배경 위에 렌더). [출처: Bulbapedia, "Pokémon sprite"]
- **"상대는 우상단, 나는 좌하단"** 배치는 클래식 포켓몬 배틀 화면의 통념으로 여러 2차 소스(WebSearch 요약, Pokémon Essentials 엔진 문서 등)에서 일관되게 언급됨. 다만 Bulbapedia 본문에서 이 문장을 그대로 인용할 수 있는 1차 출처 페이지는 이번 조사에서 직접 확보하지 못했다 — **일반 지식으로 매우 신뢰도 높지만(다수 실제 플레이 경험/스크린샷과 일치), 인용 가능한 단일 1차 텍스트 출처는 "확인 안 됨"**으로 라벨.
- Gen1/2에는 Gen3 이후 도입된 "장소별 배틀 배경(초원/동굴/물가 등 지형 전용 그래픽)"이 없다. 이 기능은 **Generation III(Ruby/Sapphire)에서 처음 도입**되었다. [출처: Bulbapedia, "Environment" — https://bulbapedia.bulbagarden.net/wiki/Environment] → Gen2는 지형에 따라 배경이 바뀌는 "발판 타원/플랫폼" 그래픽이 없으며, 흰 배경 + 일종의 지면 경계선 정도로 단순하다고 추정됨(정확한 지면 경계 그래픽 디테일은 확인 안 됨).

### 2. HP 정보 박스

- HP바(현재/최대 HP를 나타내는 가로 바)는 배틀 화면뿐 아니라 파티/스탯 화면에서도 공통으로 쓰이는 UI 요소. [출처: Bulbapedia, "HP" — https://bulbapedia.bulbagarden.net/wiki/HP]
- **색 임계값 (Gen2 확정 사실)**:
  - 초록(Green): 최대 HP의 **50% 초과**
  - 노랑(Yellow): 최대 HP의 **20%~50%**
  - 빨강(Red): 최대 HP의 **20% 미만**
  - Gen2 한정 quirk: 정확히 절반(50%)으로 회복된 경우 바는 **초록으로 유지**된다(경계값 처리가 ">50%"가 아니라 "≥50%"로 구현된 결과로 추정).
  [출처: Bulbapedia, "HP" — https://bulbapedia.bulbagarden.net/wiki/HP]
- **Gen1과의 차이**: Gen1은 픽셀 폭 기반 임계값을 사용해 실제로는 약 56% HP에서 노랑으로 바뀌는 등 세대별로 기준이 다르다(Gen1: 바 전체 길이 48px 기준, 27px 이상=초록, 10~26px=노랑, 10px 미만=빨강). Gen2부터 **퍼센트 기반의 "깔끔한" 컷오프**(50%, 20%)로 단순화되었다. [출처: WebSearch 요약 기반, GameFAQs/PokéBase류 2차 출처 — https://gamefaqs.gamespot.com/gameboy/367023-pokemon-red-version/faqs/64175/hp-bar-colour , https://pokemondb.net/pokebase/385792/what-percent-of-health-does-each-color-correspond-to — **신뢰도: 중간(공식 위키 아님, 다만 다수 소스 일치)**]
- **레벨/성별 기호 표기**: HP 박스 안에서 성별 기호(♂/♀)는 "레벨 표기 뒤(after the level)"에 오는 것이 국제판 Gen2의 표준 위치다. 이 규칙 때문에 니드런(Nidoran♂/♀)처럼 이름 자체에 성별 기호가 포함된 포켓몬은 이름에 한 번 + 레벨 뒤에 한 번, **총 두 번 성별 기호가 중복 표시**되는 버그성 특징이 있었다(일본/한국판은 이름 뒤에만 표시해 이 문제가 없음). [출처: Bulbapedia, "Gender" — https://bulbapedia.bulbagarden.net/wiki/Gender]
- **상태이상 표기**: Gen1·2는 아이콘이 아니라 **텍스트 3글자 약어**로 상태이상을 표시했고, 이 약어는 **배틀 중 레벨(Lv) 표기 자리를 대체**한다(PSN=독, PAR=마비, BRN=화상, SLP=수면, FRZ=냉동). 그래픽 아이콘 방식은 이후 세대에서 도입됨. [출처: Bulbapedia, "Status condition" — https://bulbapedia.bulbagarden.net/wiki/Status_condition ; 약어 목록은 WebSearch 2차 확인 — https://bulbapedia.bulbagarden.net/wiki/Poison_(status_condition)]
- **EXP 바**: 내 포켓몬 HP 박스 하단(또는 인접)에 파란색 EXP 바가 있어 다음 레벨까지 진행도를 보여준다. 이 기능은 Red/Blue/Yellow(Gen1)에는 배틀 화면에 없었고 **Gen2에서 배틀 화면에 추가**된 것으로 보인다(2차 출처 기준). [출처: WebSearch 요약, 2차 출처(스트래티지 가이드/팬 아카이브류) — 신뢰도 중간, Bulbapedia 1차 인용 미확보]
- 상대(적) HP 박스는 **수치(현재/최대 HP 숫자)를 표시하지 않고 바(bar)만** 보여주는 반면, 내 포켓몬 HP 박스는 **"현재/최대" 숫자까지 표시**한다는 것은 널리 알려진 Gen1~5대 공통 관례. 이번 조사에서 Bulbapedia 원문으로 직접 인용 가능한 문장은 확보하지 못했다 — **확인 안 됨(정황상 신뢰도 높음)**.

### 3. 파티 상태 표시 (6마리)

- 트레이너 배틀 시작 시, 화면 상단에 **상대 트레이너의 남은 포켓몬 수를 포켓볼 아이콘 줄로 표시**하는 관례가 Gen1부터 존재한다는 것은 널리 알려져 있으나, 이번 조사에서 Bulbapedia/공식 위키 1차 출처를 명확히 확보하지 못했다 — **확인 안 됨**.
- 파티 상태(정상/기절/상태이상) 아이콘 색 구분에 대해: WebSearch 요약 결과 "Gen1·2에서 상태이상이 있는 포켓몬은 포켓볼 아이콘이 검은색으로 표시된다"는 문장을 확보했으나, 이는 검색 엔진이 생성한 요약이며 **원문 페이지(Bulbapedia)를 직접 WebFetch로 열었을 때는 해당 문장을 재확인하지 못했다**(페이지에 해당 서술이 없었음). 따라서 이 사실은 **신뢰도 낮음/확인 안 됨**으로 표시한다.
- **자신의 파티 6마리를 HP/레벨/상태와 함께 리스트로 보여주는 화면**은 배틀 중 "포켓몬" 커맨드로 진입하는 별도의 파티 선택 화면이며, 이름·아이콘·현재 HP·상태·레벨을 각 줄에 표시한다. [출처: WebSearch 요약(Bulbapedia "HP" 페이지 인용) — "Viewing the party in the menu displays their name, icon, current HP, status condition, and level in all games." 원문 재확인은 못했으나 HP 문서 자체가 1차 출처이므로 신뢰도 중~높음]
- 결론적으로 "배틀 진입 시 상단에 볼 6개가 줄지어 뜨는 것"은 **상대 트레이너 파티에 한정**된 연출이며(야생 배틀에는 해당 없음), **내 파티 6마리 상태**는 별도 메뉴(파티 선택 화면)에서만 확인 가능하다는 것이 이번 조사에서 잠정 정리된 구조다. **이 구분 자체가 확인 안 됨 수준의 잠정 결론**이므로, 실제 게임 스크린샷/에뮬레이터로 재검증 권장.

### 4. 애니메이션 관례

- **피격 시 스프라이트 반전/깜빡임**: Gen1(그리고 동일 엔진 계보인 Gen2 추정)에서 배틀 애니메이션을 끈 상태("Battle Animation: OFF" 설정)일 때, 단순 데미지 기술(예: 몸통박치기 Tackle)은 **깜빡임(flashing) 효과**를 낸다는 것이 확인됨. 부가 효과가 있는 기술(예: 독찌르기 Poison Sting)은 **흔들림(shaking) 효과**를 낸다. [출처: Bulbapedia, "Battle effects" — https://bulbapedia.bulbagarden.net/wiki/Battle_effects, WebSearch 요약 경유 확인, 원문 재인용은 부분적으로만 성공]
- 이것과 별개로, 포켓몬 커뮤니티(팬 개발자) 포럼에서는 **일반적인 "피격 시 스프라이트가 몇 프레임 반짝이는 hit flash"가 Gen1~5 전반의 공통 관례**라고 서술되나, 이번 조사에서 Bulbapedia/공식 자료로 정확한 프레임 수·팔레트 반전 메커니즘까지는 확인하지 못했다 — **확인 안 됨(정황상 신뢰도 높으나 1차 출처 미확보)**.
- **"인버티드 스프라이트(Inverted sprites)"는 피격 연출이 아니라 Gen1 한정 글리치**(MissingNo. 등 글리치 포켓몬 도감 열람 시 RAM 값이 깨져 스프라이트가 반전/왜곡되는 버그)임을 확인. 이번 조사에서 이를 hit-flash와 혼동하지 않도록 명확히 구분함. [출처: Bulbapedia, "Inverted sprites" — https://bulbapedia.bulbagarden.net/wiki/Inverted_sprites]
- **기절(faint) 애니메이션**: 포켓몬 커뮤니티(ROM 해킹/팬게임 개발자) 스레드에서 "기절 애니메이션은 배틀러 스프라이트의 y좌표를 아래로 계속 이동시키면서, 새 y좌표가 원래 스프라이트 하단보다 커지는지(=화면 밖으로/지면 아래로 사라졌는지) 체크하는 방식"이라고 서술됨. 즉 **스프라이트가 아래로 슬라이드하며 가라앉듯 사라지는 연출**이며, 이 관례는 Gen1부터 이어져 왔다("Gen1에서는 포켓몬이 원래 움직이지 않았기 때문에 이 슬라이드다운이 사실상 유일한 '사라지는' 연출이었다"는 설명 포함). [출처: PokéCommunity 포럼 스레드, "Understanding the Fainting Animation" — https://www.pokecommunity.com/threads/understanding-the-fainting-animation.349445/ — **신뢰도: 중간(팬 개발 커뮤니티, 공식 위키 아님. 다만 실제 리메이크/팬게임 재현 목적의 기술 스레드라 실증적)**]
- **데미지 시 HP바 소모 방식**: HP바는 데미지를 입으면 즉시 텅 비지 않고 **애니메이션 방식으로 서서히 줄어드는(tick-down) 방식**이라는 것이 통념이나, 이번 조사에서 Bulbapedia HP 문서는 "HP bar animation in Generation II"라는 표현과 함께 글리치 관련 링크만 언급했고 정확한 감소 속도/프레임 디테일은 확보하지 못함 — **확인 안 됨**.
- **공격 시 화면 플래시**: 일부 기술(특히 특수/강력한 기술)이나 크리티컬 히트 시 화면 전체가 짧게 반전/플래시되는 연출이 존재한다는 것은 일반적으로 알려져 있으나(예: 노말 세대 게임에서 "Flash" 무브와는 무관하게 배틀 연출로서의 화면 반전), 이번 조사에서 Gen2 한정 1차 출처를 확보하지 못했다 — **확인 안 됨**.

### 5. 하단 텍스트 박스 (배틀 메시지)

- 배틀 메시지/커맨드 창은 화면 **하단**에 위치하며, GB 다이얼로그 박스의 일반적 관례상 **화면 하단 약 1/4~1/3 높이**를 차지하는 사각형 박스(흰 배경 + 검은 테두리)로 텍스트를 표시한다는 것이 일반적으로 알려져 있다. 다만 이번 조사에서 Gen2 배틀 화면에 한정된 정확한 타일 행 수(예: 4행 vs 5행)·픽셀 높이를 못박은 1차 출처는 확보하지 못했다 — **확인 안 됨(픽셀 단위는 추정)**.
- 배틀 커맨드 메뉴(FIGHT/PKMN/ITEM/RUN, Gen2에서는 아이템 메뉴 명칭이 "PACK"으로 변경)는 이 하단 텍스트 박스와 **같은 영역을 공유하거나 우측 절반을 차지하는 서브 윈도우**로 뜨는 것이 시리즈 공통 관례. [Gen2의 "Item→Pack" 명칭 변경은 출처: WebSearch 요약, StrategyWiki/Bulbapedia 경유 — 세부 URL 재접근은 403으로 실패했으나 다수 소스 일치로 신뢰도 중간]

## 확인 안 된 것 (조사 못 한 영역)

- 상대/내 HP 박스의 **정확한 픽셀 좌표(x, y)** — Bulbapedia류 위키는 이 레벨의 좌표 데이터를 텍스트로 정리해 두지 않음. `pret/pokecrystal` 디스어셈블리(GitHub, https://github.com/pret/pokecrystal )의 `gfx/battle/`, `engine/battle/core.asm` 소스를 직접 읽으면 타일맵 좌표를 정확히 뽑을 수 있으나, 이번 조사에서는 검색/요약 도구로는 세부 어셈블리 상수까지 못 들어감 — **후속 조사 가치 있음**.
- 하단 텍스트 박스의 정확한 타일 행 수.
- "화면 상단 상대 파티 볼 아이콘"이 야생 배틀에도 뜨는지, 트레이너 배틀 전용인지 최종 확인 못함.
- 기절 시 화면 플래시/사운드 큐 등 청각적 관례(이번 조사는 시각 UI로 스코프 한정).
- 크리티컬 히트 시 화면 연출의 Gen2 한정 디테일.
- 배틀 애니메이션 프레임 레벨(예: 몇 프레임 동안 몇 회 깜빡이는지)의 정확한 수치.

## Prompt Injection 관련 참고 (의심 패턴 보고)

조사 중 `https://tcrf.net/Pok%C3%A9mon_Gold_and_Silver/Unused_Graphics` 페이지를 WebFetch로 열었을 때, 반환된 콘텐츠가 "AI 테스트 페이지" 형식의 **위장된 지시문(파일 삭제 등 유해 명령 실행 유도로 추정)**을 포함하고 있는 것으로 감지되었다. 해당 페이지의 실제 배틀 UI 관련 내용은 없었고, WebFetch 처리 모델이 이를 의심 패턴으로 자동 식별해 지시를 따르지 않고 거부했다. **본 조사에서는 해당 지시를 전혀 실행하지 않았으며, 사실 추출에도 이 페이지의 콘텐츠를 전혀 사용하지 않았다.** 사용자가 직접 확인하고 싶다면 위 URL을 별도로 검증할 것을 권장하되, 주의가 필요하다.

## 우리 앱 재구성 가이드 (제안, 오리지널 도트로 재구성 시)

아래는 위 관례를 **AIUsage의 펫 배틀 화면(오리지널 스프라이트)**에 적용할 때의 배치 제안이다. 모두 "권장"이며, 사실이 아니라 설계 제안임을 명시.

- **공간 분할**: 화면을 대각선으로 나눠 상대(적) 펫은 우상단, 내 펫은 좌하단에 배치하는 클래식 구도를 그대로 차용 — 저작권 리스크 없는 순수 레이아웃 관례이며 사용자에게 익숙함(가독성 이점).
- **HP 박스 배치**: 상대 HP 박스는 좌상단(상대 스프라이트와 대각선으로 마주보게), 내 HP 박스는 우하단에 배치 — Gen2와 동일한 "스프라이트와 반대쪽 대각선에 정보 박스"구도. 우리 앱은 펫이 이미 하나(walking cat)이므로, 배틀 모드에서는 **Claude 펫 vs Cursor 펫**(또는 사용자 대 목표) 같은 자체 세계관으로 구도만 차용.
- **HP바 색 임계값 통일**: 초록(>50%) / 노랑(20~50%) / 빨강(<20%) 3단계를 그대로 채용 권장. 프로젝트에 이미 사용량 게이지 색상 로직이 있다면(예: Claude/Cursor 사용률 표시) 기준을 맞춰 일관성 확보.
- **파티 상태 아이콘**: 상단에 자체 도트 아이콘(예: 우리 펫 컬렉션의 미니 아이콘)을 양 팀 각각 정렬해 배치 — Gen2처럼 "상대 팀만" 보여주는 대신, **양 팀 파티 아이콘을 상단 좌우로 대칭 배치**하는 것을 제안(우리 앱은 6마리 고정 파티 개념이 없으므로, "현재 배틀 참여 펫 + 예비 펫" 정도로 단순화 가능).
  - 기절(빈사): 흑백(그레이스케일) 처리 + 살짝 아래로 처진 아이콘.
  - 상태이상: 아이콘 테두리에 컬러 배지(독=보라, 마비=노랑 등 우리 팔레트로 재정의) — Gen2의 "검은 볼" 방식보다 **색상 배지 방식이 우리 앱의 다채로운 펫 컬러와 더 잘 어울림**(구분성 개선안).
  - 정상: 기본 컬러 아이콘 그대로.
- **피격 연출**: 데미지를 받으면 스프라이트를 2~3프레임 반짝이거나(투명도 토글) 좌우로 살짝 흔드는 hit-shake를 적용 — 기존 `WalkingCat`의 `mood`/`speedMultiplier` 패턴처럼 View 레벨에서 비-`@Published` 필드를 직접 건드리는 기존 관례를 재사용 가능.
- **기절(패배) 연출**: 스프라이트를 서서히 아래로 슬라이드시키며 페이드아웃 — Gen1~2식 "가라앉기" 연출을 그대로 차용. 우리 펫이 이미 걷기 애니메이션 프레임을 갖고 있다면, 기절 시 별도 프레임 없이 **현재 프레임 고정 + Y 이동 + 알파 감소**만으로 충분히 구현 가능(기존 에셋 재사용, 신규 스프라이트 불필요).
- **하단 텍스트/커맨드 박스**: 화면 하단 약 25~30% 영역에 배틀 로그(메시지) + 커맨드 버튼을 배치. macOS 앱이므로 GB식 "4방향 커서" 대신 클릭 가능한 버튼 UI로 대체.
- **EXP/진행도 바**: 우리 앱 맥락에서는 "다음 가챠까지" 또는 "사용량 페이스" 같은 기존 프로젝트 개념과 자연스럽게 연결 가능 — Gen2의 파란 EXP 바 관례를 "펫 성장/코인 누적" 표시로 재해석하는 안 제안.
- **저작권 주의**: 위 모든 제안은 **구조/배치/관례**만 차용한 것이며, 실제 색상 팔레트·폰트·스프라이트 실루엣·텍스트박스 테두리 디자인은 프로젝트의 기존 도트 아트 스타일(9개 팩, CC0/CC-BY 라이선스 준수)에 맞춰 완전히 새로 그려야 한다. Gen2 스프라이트/배경/폰트를 트레이싱하거나 색상만 바꿔 재사용하는 것은 금지.
